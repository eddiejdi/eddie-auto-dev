# Kiosk Dashboard — Diagnóstico e Correções (2026-05-12)

**Servidor**: `homelab@192.168.15.2`  
**Hardware**: Intel UHD 630 (onboard), saída `HDMI-3`, `1920x1080@60Hz`  
**Serviço principal**: `homelab-dashboard.service`  
**Browser kiosk**: `surf` (webkit2gtk) via `tty1`, display `Xorg :0`

---

## Problemas identificados e correções aplicadas

### 1. Output X11 primário desconectado (`DP-5 disconnected primary`)

**Sintoma**: kiosk não aparecia no monitor físico, tudo rodando normalmente em software.  
**Causa**: ao iniciar o Xorg, o output `DP-5` (sem cabo) estava marcado como `primary`. O `surf` renderizava para essa saída sem sinal. O monitor onboard (`HDMI-3`) estava conectado mas não era primário.

**Diagnóstico**:
```bash
sudo -u homelab env DISPLAY=:0 XAUTHORITY=/home/homelab/.Xauthority xrandr --query
# DP-5 disconnected primary
# HDMI-3 connected 1920x1080+0+0
```

**Correção** — adicionado em `/usr/local/bin/tty1-wrapper` logo após o `xset`:
```bash
# Forçar output conectado como primário
CONNECTED_OUT=$(xrandr | awk '/ connected/ {print $1; exit}')
if [ -n "$CONNECTED_OUT" ]; then
    xrandr --output "$CONNECTED_OUT" --primary --auto
fi
```

---

### 2. Processos órfãos (`left-over processes`) após restart

**Sintoma**: `systemctl restart homelab-dashboard` deixava `Xorg`, `openbox`, `xinit` anteriores ainda rodando. O novo start herdava display sujo e o `surf` não abria.

**Causa**: `KillMode=process` (padrão) mata apenas o PID principal do serviço, não os filhos.

**Diagnóstico**:
```
homelab-dashboard.service: Unit process 1276413 (xinit) remains running after unit stopped.
homelab-dashboard.service: Found left-over process 1276414 (Xorg) in control group while starting unit.
```

**Correção** — criado `/etc/systemd/system/homelab-dashboard.service.d/killmode-fix.conf`:
```ini
[Service]
KillMode=control-group
KillSignal=SIGTERM
TimeoutStopSec=10
ExecStop=/bin/bash -c 'pkill -f "Xorg :0" 2>/dev/null; pkill -f "xinit.*:0" 2>/dev/null; pkill -x surf 2>/dev/null; pkill -x openbox 2>/dev/null; pkill -x unclutter 2>/dev/null; true'
```
```bash
sudo systemctl daemon-reload
```

---

### 3. Tela de login do Grafana no kiosk

**Sintoma**: surf abria a URL do Grafana e aparecia a tela de login OAuth (Authentik).  
**Causa**: container Grafana não tinha `GF_AUTH_ANONYMOUS_ENABLED=true` ativo (variável estava no compose mas o container rodava com vars antigas de um `docker run` manual anterior).

**Diagnóstico**:
```bash
curl -sI "http://localhost:3002/d/homelab-btop/..." | grep HTTP
# HTTP/1.1 302 Found → Location: /login?redirectTo=...
docker inspect grafana --format "{{range .Config.Env}}{{println .}}{{end}}" | grep -i anon
# (nenhuma saída)
```

**Correção**: recriar container com vars completas (ver seção abaixo). O `docker-compose.grafana.yml` já continha as vars corretas — o container estava dessincronizado.

---

### 4. No Data nos dashboards após recreate do container

**Sintoma**: após recriar o container Grafana, todos os painéis mostravam "No Data".  
**Causa**: o container foi recriado na rede `bridge` genérica. O datasource Prometheus usa o hostname `prometheus` (nome do container), que só é resolvível dentro da rede `homelab_monitoring`.

**Diagnóstico**:
```bash
docker exec grafana wget -qO- "http://prometheus:9090/api/v1/query?query=up"
# (falha silenciosa)
docker inspect grafana --format "{{json .NetworkSettings.Networks}}"
# {"bridge": {...}}  ← errado, deveria ser homelab_monitoring
docker inspect prometheus --format "{{json .NetworkSettings.Networks}}"
# {"homelab_monitoring": {...}}
```

**Correção**: recriar container com `--network homelab_monitoring`.

## Comando completo de recreate correto do Grafana

```bash
docker stop grafana && docker rm grafana
docker run -d \
  --name grafana \
  --restart unless-stopped \
  --network homelab_monitoring \
  -p 3002:3000 \
  -v /home/homelab/monitoring/grafana/provisioning:/etc/grafana/provisioning \
  -v /home/homelab/monitoring/grafana/data:/var/lib/grafana \
  -e GF_SECURITY_ADMIN_PASSWORD="<senha>" \

  -e GF_AUTH_GENERIC_OAUTH_ALLOW_SIGN_UP="true" \
  -e GF_AUTH_GENERIC_OAUTH_CLIENT_ID="authentik-grafana" \
  -e GF_AUTH_GENERIC_OAUTH_CLIENT_SECRET="<secret>" \

  grafana/grafana:12.4.0
```
